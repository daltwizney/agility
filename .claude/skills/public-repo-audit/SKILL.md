# Public Repo Audit

This is a public open-source repo. Run a full audit of every git-tracked file against the rules in `CLAUDE.md` → `Public Repo Safety`. **That section is the source of truth — re-read it before you start**, since the rules may have evolved since this skill was written.

## Procedure

1. **Enumerate scope**: `git ls-files`. Audit only tracked files. (If something *should* be `.gitignore`'d but is tracked, that itself is a finding.)

2. **Read every text file in full** with the Read tool. Don't rely on grep alone — patterns miss multi-line context and obfuscated cases. For binary / LFS-tracked files (`*.umap`, `*.uasset`, images, audio, video, fonts, etc.), note their presence but don't try to inspect content; flag only if the filename suggests something sensitive (e.g. `desktop-screenshot-*.png`, `creds.png`).

3. **Check each file against the CLAUDE.md `Public Repo Safety` rules**:
   - User-home absolute paths (allowed exception: `/Users/Shared/...`)
   - Secret-shaped content (`sk-…`, `ghp_…`, `gho_…`, `AKIA…`, `xoxb-…`, `-----BEGIN … PRIVATE KEY-----`, JWTs, long random tokens, `.env`-style `KEY=value` lines with secret-shaped values)
   - Personal identity (emails, full names, GitHub handles, Apple IDs, hostnames, MAC addresses, local IPs like `192.168.x.x` / `10.x.x.x` / `172.16-31.x.x`)
   - Build artifacts that leak local paths and shouldn't be tracked (anything under `Binaries/`, `Intermediate/`, `Saved/`, `DerivedDataCache/`, `.idea/`, `.vs/`; generated IDE projects like `*.xcodeproj`, `*.xcworkspace`, `*.sln`)
   - Hooks/scripts that auto-execute on clone or startup without obvious intent (Claude Code hooks, git hooks, `postinstall` scripts, etc.)
   - **Python-specific** (apply to `python/` and any other `.py` / `.ipynb` / `pyproject.toml` files; mirrors the "Python-side specifics" block in CLAUDE.md):
     - Tracked `.env` files — only sanitized `.env.example` should ever be tracked.
     - Tracked model weights / large binaries: file extensions `.pt`, `.pth`, `.onnx`, `.safetensors`, `.gguf`, `.bin`, `.h5`, `.ckpt`, `.pkl` (belong in a download-on-first-run cache, not the repo).
     - Server bind addresses to `0.0.0.0` in tracked Python code (`host="0.0.0.0"`, `--host 0.0.0.0`, uvicorn / FastAPI startup args, etc.) — flag for human review; almost always accidental.
     - Jupyter notebooks (`.ipynb`) with non-empty `outputs` arrays — output cells leak absolute paths, env-var dumps, and embedded screenshots; flag for clearing before commit.
     - Python cache / build artifacts that slipped past `.gitignore`: `__pycache__/`, `*.pyc`, `*.pyo`, `.venv/`, `venv/`, `.pytest_cache/`, `.mypy_cache/`, `.ruff_cache/`, `*.egg-info/`, `.ipynb_checkpoints/`.
     - AI / ML credential shapes the generic patterns miss: `hf_…` (Hugging Face), `r8_…` (Replicate), 40-char hex following `WANDB_API_KEY=` (Weights & Biases), `MODAL_TOKEN_ID` / `MODAL_TOKEN_SECRET`.
     - Code patterns that are secure-coding red flags rather than leaks — flag for human review with severity MEDIUM: `pickle.load`/`pickle.loads`/`marshal.loads`/`shelve.open`/`torch.load(..., weights_only=False)` reading from request bodies, file uploads, or network sockets; `eval(`/`exec(`/`subprocess.run(..., shell=True)` with non-literal arguments.

4. **Honor the acknowledged exceptions** (don't flag as issues — mention as informational only if directly relevant):
   - `/Users/Shared/Epic Games/UE_<version>/...` — engine install paths are identical across machines.
   - AFS `SecurityToken=...` in `Config/DefaultEngine.ini` — auto-generated by UE; standard public-repo practice.
   - `ProjectID=...` in `Config/DefaultGame.ini` — UE-generated project GUID, not a secret.
   - For any new auto-generated `Config/*.ini` identifier, apply the same test: **project-scoped + dev-time-only + auto-regenerated = OK; user/account-scoped + grants real access = not OK**.

5. **Report findings**, grouped by severity:
   - **HIGH**: real secrets, user-home absolute paths in tracked files, personal credentials — anything that would matter if a stranger cloned the repo.
   - **MEDIUM**: personal identity where placeholders should be used, machine-local state, would-be-`.gitignore`'d artifacts being tracked.
   - **LOW / informational**: acceptable exceptions, UE-generated identifiers, anything where practical exploit cost is essentially infinite.

   For each finding, give: `file:line`, what was found (**redact partial values** for any real-looking secret — e.g. `sk-ant-…████████`), why it's a concern, recommended action. End the report with a short "Clean" list of files that were audited and had no findings, so coverage is visible.

6. **Don't fix anything.** Report only. The user reviews and decides what to remediate. Per CLAUDE.md, never stage changes.
